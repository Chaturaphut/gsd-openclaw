# Contributing to GSD-OpenClaw

Thanks for your interest in contributing! 🎉

## How to Contribute

### 🐛 Report Bugs
Open an [issue](https://github.com/Chaturaphut/gsd-openclaw/issues) with:
- Clear description of what went wrong
- Steps to reproduce
- Expected vs actual behavior

### 💡 Suggest Features
Open an issue tagged `enhancement` with:
- Use case / problem description
- Proposed solution
- How it fits the multi-agent workflow

### 📝 Improve Documentation
- Fix typos, improve clarity, add examples
- PRs welcome — no issue needed for docs-only changes

### 🔧 Submit Changes

1. Fork the repo
2. Create a feature branch: `git checkout -b feat/your-feature`
3. Make your changes
4. Follow [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`, `docs:`
5. Submit a Pull Request

### Branch Naming
| Prefix | Use When |
|--------|----------|
| `feat/` | New feature |
| `fix/` | Bug fix |
| `docs/` | Documentation |
| `refactor/` | Code restructure |

## Style Guide

- **Markdown:** Use clear headers, bullet points, and code blocks
- **Templates:** Keep them generic — don't add project-specific details
- **Workflow docs:** Include "When to use" and "Who does it" sections
- **Examples:** Show realistic but simple scenarios

## Cherry-Picking from Upstream GSD

When adapting features from [GSD](https://github.com/gsd-build/get-shit-done):

1. Document which GSD version the feature comes from
2. Adapt for multi-agent context (not single-developer)
3. Add role assignments (who does what)
4. Include OpenClaw-specific integration notes
5. Update CHANGELOG.md

## Code of Conduct

Be kind, be constructive, and respect everyone's time. We're all here to build better tools.

Thank you for helping make AI agent workflows better! 💜
